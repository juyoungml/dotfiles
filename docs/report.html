Dotfiles renewal · July 2026

Small enough to understand. Strong enough to rebuild a machine.

A research-backed plan for a public, fast, secure, uv-first and agent-native development environment.

Recommendation: use a normal public Git repository with a tiny idempotent deployer today. Keep source files manager-independent, so Chezmoi remains an easy upgrade only when real templating pressure appears.

Authentication: remove friction, not security

GitHub

Use gh auth login --web --git-protocol ssh once. GitHub CLI stores OAuth in the OS credential store when available. Do not globally export GH_TOKEN; it overrides stored auth for every child process.

Hugging Face

Use hf auth login with browser/device login. For headless automation, scope HF_TOKEN narrowly and use least privilege.

1Password

Use desktop biometric integration and op run for one command. Keep only op:// locators in reference files; even locator names can reveal metadata.

Important: op run places plaintext in a child environment. Other processes under the same OS user may observe it. Never wrap the whole shell, IDE, or agent session.

Agent-native without publishing your life

LayerPublic repositoryPrivate/local
InstructionsStable principles: minimal changes, teach reasoning, verify, preserve unrelated workEmployer/client constraints and private machine rules
SkillsReusable, sanitized SKILL.md workflowsInternal URLs, account IDs, proprietary examples
MemoryPolicy and schema templates onlyRaw observations, auto-memory, project context, transcripts
SecretsAt most safe op:// referencesSecret manager only; never agent memory

The durable loop is: local observation → repetition or explicit approval → review → public instruction, reusable skill, or private fact. This prevents one-off agent guesses from becoming permanent taste.

Public bootstrap and trust

A memorable juyoungml.site/bashrc route is useful, but a custom domain does not make mutable remote code safer. Prefer a two-step inspect/run path, and offer the pipe only as a convenience:

curl -fsSLo /tmp/dotfiles https://juyoungml.site/bashrc
less /tmp/dotfiles
sh /tmp/dotfiles --profile dev

# Convenient, but executes mutable remote code:
curl -fsSL https://juyoungml.site/bashrc | sh -s -- --profile dev

For stronger integrity, use an immutable GitHub release, pin a tag, publish SHA-256 checksums, and verify assets. Before making the repository public, scan the full Git history and rotate any exposed credential before history rewriting.

What was intentionally removed

  • Poetry, its uninstall script, and global pip aliases: uv now owns the Python workflow.
  • Miniconda base-shell initialization: it overlaps with uv and adds startup mutation.
  • Unpinned CUDA/NVIDIA/NodeSource installers: host-specific infrastructure should not live in a universal bootstrap.
  • Oh My Zsh/theme dependence and Git work on every prompt: useful features can return only when their recurring value exceeds their cost.
  • Competing VS Code formatters, themes, icon packs, and AI extensions: recommendations are now a small language/tooling baseline.
